# 03 环境与部署全流程

## 一键路径（推荐）

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
make setup
```

`make setup` 实际做四件事：

1. 检查 Docker daemon  
2. 自动准备 `.env`（若缺失则从 `.env.example` 复制）  
3. 预拉取镜像  
4. `make up` 拉起服务并执行健康等待

## 启动与验证关键命令

- `make up`：启动并等待健康
- `make verify`：运行 `scripts/verify-setup.py`，输出 PASS/FAIL/TIMEOUT 表
- `make pipeline`：跑通采集到训练全链路

## `verify` 的检查项

- MinIO：health endpoint + bucket 是否存在
- PostgreSQL：连接可用 + `hive_metastore` / `mlflow` DB 存在
- Hive Metastore：9083 端口 TCP 可达
- Trino：`/v1/info` 可用且 `starting=false`
- MLflow：`/health` 返回 200

## 常见失败与处理

### 1) `make wait` 超时

- 先执行 `make verify` 看是哪项 FAIL/TIMEOUT。
- 常见原因：Hive 启动慢、环境变量缺失、端口冲突。

### 2) API 相关失败

- ECB 接口可能波动，pipeline 有采集重试机制。
- 必要时重新执行 `make pipeline`。

### 3) Python 工具命令找不到

- 使用 `make PYTHON=.venv/bin/python <target>` 保证使用虚拟环境解释器。

## 面试表达建议

“我把部署体验做成了可脚本化的标准入口，不只是能跑起来，还能自证健康状态。`verify` 不是简单 ping 端口，而是检查服务语义级状态（例如 bucket 和数据库实体）。”
