# 15 高频面试题库（平台工程师 100k+）

## 架构类

### Q1: 为什么 v1 不直接上 K8s？

**答题要点**
- 项目定位是 reference implementation，不是生产集群产品。
- Compose 能确保低门槛复现，适合验证流程与工程模式。
- v3 规划再上 K8s/Terraform，演进路径清晰。

### Q2: 你如何保证数据链路可靠？

**答题要点**
- 采集重试 + 明确失败语义（`StepError`）。
- Bronze 质量检查与 rejected records 隔离坏数据。
- `verify` + CI 门禁 + pipeline 日志形成验证闭环。

## 数据工程类

### Q3: 为什么用 Medallion？

- Bronze 保留原始可追溯，Silver 清洗标准化，Gold 面向消费。
- 通过层次隔离复杂度，避免“一个表做所有事”。

### Q4: 如何避免时间序列泄露？

- 训练阶段使用 `TimeSeriesSplit`，按时间顺序验证。
- 特征构造严格用事件前后可用窗口，不用未来值。

## 平台工程类

### Q5: 你如何做发布质量控制？

- 本地：`setup -> verify -> pipeline -> lint/typecheck/test`。
- CI：lint -> typecheck -> test(coverage gate)。
- 发布：tag + release + checklist + history 追踪。

### Q6: 你如何规划 v2？

- 引入资产编排与调度，保留 legacy 脚本做迁移缓冲。
- 先解决可观测和可重试语义映射，再做全面替换。

## 行为类（结合项目）

### Q7: 你遇到过最难的问题是什么？

建议回答：
- 描述启动依赖与健康检查不一致导致的“看似启动成功但不可用”问题。
- 说明如何通过 `service_healthy + wait + verify` 形成系统级修复。

### Q8: 如何证明你不是“只会写代码”？

- 展示完整工程闭环：编码、测试、发布、文档、演进与决策记录。
- 强调 history 文档体系保障团队长期可维护。
