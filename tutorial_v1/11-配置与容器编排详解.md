# 11 配置与容器编排详解

## `docker/docker-compose.yml` 关键点

- 全部服务配置健康检查。
- `depends_on: condition: service_healthy` 保证启动顺序。
- MinIO 初始化由一次性 `minio-init` 容器完成（幂等建桶）。

## 关键服务角色

- MinIO：对象存储 + MLflow artifacts
- PostgreSQL：Hive Metastore + MLflow backend
- Hive Metastore：表元数据
- Trino：查询引擎（`hive` + `iceberg` 两个 catalog 模板）
- MLflow：实验管理
- **可选（v2.5）**：`docker-compose.openmetadata.yml` + `make up-openmetadata` 启动 OpenMetadata（Elasticsearch + OM MySQL + 迁移作业 + 服务），用于元数据发现；不占默认 `make up`。

## 配置模板化

- `config/trino/catalog/hive.properties` 使用 `${VAR}` 占位。
- `scripts/trino-entrypoint.sh` 在启动时 `envsubst` 注入环境变量。

## 环境变量策略

- `.env.example` 提供可运行默认值。
- `verify-setup.py` 校验必需变量，降低“隐性错配”。

## 面试亮点

“我没有把配置硬编码进代码，而是走环境变量 + 模板注入，这让本地、CI、未来多环境部署能保持同一套配置语义。”
