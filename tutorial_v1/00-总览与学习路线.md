# 00 总览与学习路线

## 你要达到的面试能力

完成本教程后，你需要能稳定完成三种讲解模式：

1. **10 分钟全景介绍**：问题、架构、实现、结果、演进。
2. **30 分钟深挖问答**：可靠性、数据质量、CI、部署、取舍。
3. **系统设计延展**：从 v1 走到 v2/v3 的迁移策略与风险控制。

## 项目一句话定义

SoloLakehouse 是一个单机 Docker Compose 形态的 Lakehouse 参考实现：从数据采集到 Bronze/Silver/Gold，再到 SQL 查询和 ML 实验跟踪，展示“平台工程化”最小闭环。

## 核心能力地图

- **架构能力**：知道五层核心与八层目标的关系。
- **实现能力**：能说清 `ingestion -> transformations -> ml` 的关键函数与 I/O 路径。
- **工程能力**：会解释 `make setup/up/verify/pipeline`、测试与 CI 质量门禁。
- **运维能力**：能快速定位服务启动、数据缺失、接口超时、实验不出结果等问题。
- **演进能力**：能解释为什么 v2 走 Dagster，v3 才上 K8s/Terraform。

## 学习节奏建议

### 第 1 轮（理解）

- 阅读 `01-04`，建立业务与架构语境。
- 目标：能画出数据流与服务依赖图。

### 第 2 轮（源码）

- 阅读 `05-12`，结合代码路径逐段验证。
- 目标：每个模块至少能说出 3 个关键实现点与 2 个边界条件。

### 第 3 轮（面试输出）

- 阅读 `13-18`，形成固定话术和高频问答。
- 目标：面对追问仍能结构化表达，不陷入“只会背文档”。

## 高薪面试官通常关注什么

- 你是否理解“为什么这样设计”，而不是只会说“我用了什么组件”。
- 你是否考虑失败路径、恢复路径、可验证性。
- 你是否有可落地的未来演进路线，并且知道迁移风险。

## 你在面试中应避免

- 把规划中的 v2/v3 当作已实现能力来讲。
- 只讲技术名词，不讲取舍依据。
- 忽略成本、复杂度、可维护性和团队可执行性。
