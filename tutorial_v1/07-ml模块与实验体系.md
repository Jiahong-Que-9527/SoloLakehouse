# 07 ML 模块与实验体系

## 文件与职责

- `ml/train_ecb_dax_model.py`：训练函数与模型选择。
- `ml/evaluate.py`：参数网格实验、MLflow 记录、最佳 run 输出。

## 训练策略

- 任务：预测 `dax_return_1d > 0` 的二分类。
- 模型：XGBoost / LightGBM 二选一。
- 验证：`TimeSeriesSplit(n_splits=5)`，避免时间泄露。

## 关键特征

- `rate_change_bps`
- `rate_level_pct`
- `is_rate_hike`, `is_rate_cut`
- `dax_volatility_pre_5d`
- `dax_pre_close`

## 实验网格

- `model_type`: `xgboost`, `lightgbm`
- `n_estimators`: `50, 100, 200`
- `max_depth`: `3, 5`

共 12 组实验，指标写入 MLflow，并上传模型 artifact。

## 面试关注点

### 为什么使用 TimeSeriesSplit

- 随机切分会把未来信息泄漏到过去。
- 时间序列任务必须保持时间顺序评估。

### 当前局限

- 特征量较少，样本规模有限，性能上限受限。
- v2/v3 可扩展方向：更多 macro feature、滚动窗口特征、模型注册与线上 serving。
